Conceitos Básicos de Git e Git Hub

O Que São Eles?

A função do Git e do GitHub é proporcionar a facilitação do "versionamento de códigos", mas eles possuem também inumeras funções tanto particulares e pessoais, quanto corporativas que facilitam a vida do profissional desenvolvedor.

Principais Funções:

  • Hospedagem de Repositórios
  • Colaboração em Equipes
  • Rastreamento de Problemas e Projetos
  • Criação de Forks e Branches
  • Versionamento de Códigos

Configurando o Git

Abra o Git Bash na sua pasta de destino das informações e siga as seguintes etapas:

  • $git config --global user.name "nome_do_usuario"
  • $git config --global user.email "email@dousuario"

Esses comandos servem para configurar as informações globais do usuário. Caso erre em algo, basta dar o comando novamente.

Principais Comando do Git:

  • git init: Inicia um novo repositório Git no diretorio atual.
  • git clone + URL: Clona um repositório existente para o diretório local.
  • git add .: Adiciona alterações no índice e prepara essas alterações para o commit.
  • git commit -m "mensagem": Cria um commit com as alterações adicionadas, incluindo a mensagem que descreve a mudança.
  • git status: Descreve o status atual do repositório, mostrando as adições, exclusões e modificações.
  • git log: Mostra o histórico de commits no repositório.
  • git branch [nome_da_branch]: Cria uma nova branch.
  • git branch: Lista todas as branches locais e destaca a atual.
  • git checkout [nome_da_branch]: Altera para uma branch específica.
  • git merge [branch]: Combina as branchs dentro da branch atual.
  • git pull: Atualiza o repositório LOCAL com os dados do repositório REMOTO.
  • git push [remote] [branch]: Envia os commits locais para o repositório remoto.
  • git remote -v: Lista os repositórios remotos configurados.
  • git fetch: Recupera as alterações do repositório remoto, mas não faz um merge automático.
  • git reset [arquivo]: Ele desfaz as alterações feitas no arquivo especificado.
  • git rm [arquivo]: Remove um arquivo do repositório e inclui no próximo commit.
  • git diff: Mostra as diferenças entre as alterações não adicionadas.
  • git remote add [nome_do_rep] [URL]: Adiciona um repositório remoto com nome especificado.
  • git push add origin main: Usado para fazer push's de repositório local para o remoto.
